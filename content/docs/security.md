# 安全

Pi 是一个本地编码代理。它使用启动它的用户账户所拥有的权限运行，并将该用户可写的文件视为处于同一个本地信任边界内。

## 项目信任

项目信任控制 pi 是否加载项目本地的设置、资源、包和扩展。它不是沙箱，也不会限制你开始在某个目录中工作后，模型可要求工具执行哪些操作。

当 pi 从当前工作目录发现以下任一内容时，会认为项目含有需要信任的资源：

- `.pi/settings.json`
- `.pi/extensions`、`.pi/skills`、`.pi/prompts` 或 `.pi/themes`
- `.pi/SYSTEM.md` 或 `.pi/APPEND_SYSTEM.md`
- 当前目录或祖先目录中的项目 `.agents/skills`

仅有一个空的 `.pi` 目录，不算作需要信任的项目资源。

当交互会话在含有需要信任的资源的项目中启动，且当前目录或父目录没有已保存的决定时，pi 会遵循全局设置中的 `defaultProjectTrust`。默认值是 `"ask"`，即在 UI 可用时询问是否信任该项目。保存的决定按规范化目录存储在 `~/.pi/agent/trust.json` 中，并且在应用全局默认值之前，会采用当前路径或父路径上距离最近的已保存决定。

信任项目将允许 pi 加载需要信任的项目资源，包括：

- `.pi/settings.json`
- `.pi` 资源，例如扩展、技能、提示词模板、主题和系统提示词文件
- 通过项目设置配置但尚未安装的项目包
- 项目本地扩展和由项目包管理的扩展

拒绝信任会跳过受保护的资源。除非禁用了上下文加载，否则无论项目信任状态如何，都会加载 `AGENTS.md` 和 `CLAUDE.md` 上下文文件。在信任状态确定前，pi 只加载上下文文件、用户/全局扩展和 CLI `-e` 扩展。用户/全局扩展和 CLI 扩展可以处理 `project_trust` 事件；第一个返回“是/否”决定的扩展将取得该决定的控制权。

非交互模式（`-p`、`--mode json` 和 `--mode rpc`）不会显示信任提示。如果没有适用的已保存信任决定，`defaultProjectTrust: "ask"` 和 `"never"` 会忽略这些资源，而 `"always"` 会信任它们。使用 `--approve`/`-a` 或 `--no-approve`/`-na` 可为单次运行覆盖项目信任设置。

## 无内置沙箱

Pi 不包含内置沙箱。内置工具可以使用 pi 进程的权限读取文件、写入文件、编辑文件和运行 shell 命令。扩展是以相同权限运行的 TypeScript 模块。包安装、shell 命令、语言服务器、测试命令和其他开发工具均与普通本地进程行为一致。

这是有意为之。Pi 旨在操作本地源代码树、调用项目工具链，并与用户现有的开发环境集成。部分式的进程内沙箱很容易被误解为安全边界，而它仍然依赖主机 shell、文件系统、包管理器、凭据和扩展代码。真正的隔离需要来自操作系统或虚拟化/容器边界。

项目信任仅是输入加载防护。它可防止仓库在你批准之前静默更改 pi 的设置或扩展，但不会使不受信任的代码、不受信任的提示词或不受信任的模型输出变得安全。来自仓库文件、注释、文档、上下文文件或构建输出的提示词注入，是本地代理的预期风险，pi 无法可靠地加以阻止。

## 运行不受信任或无人监督的工作

对于不受信任的仓库、你不打算密切监督的生成代码或无人值守的自动化，应在受控环境中运行 pi。请使用容器、虚拟机、微型虚拟机、远程沙箱或策略控制沙箱，并仅提供任务所需的文件和凭据。

[容器化](containerization.md)中记录了常见模式：

- 在容器/沙箱内运行整个 `pi` 进程
- 在主机上运行 pi，同时将内置工具执行路由到 Gondolin 微型虚拟机
- 仅挂载代理应访问的工作区路径
- 除非容器应访问主机会话、设置和凭据，否则不要挂载主机的 `~/.pi/agent`
- 只传入最低限度的必要 API 密钥，或使用短期凭据
- 当任务不需要网络时限制网络访问
- 将结果复制回受信任系统前，先审查差异和输出

如果以读写方式绑定挂载主机工作区，容器或虚拟机内部的写入仍可修改主机文件。如需更强地防范意外写入，请使用只读挂载，或在沙箱内外复制文件。

## 报告安全问题

要报告安全问题，请遵循仓库的[安全策略](https://github.com/earendil-works/pi-mono/blob/main/SECURITY.md)。请勿针对涉及安全的报告提交公开 issue。

本地代理的预期行为、缺少内置沙箱、来自不受信任内容的提示词注入，以及用户安装的扩展或技能的行为，通常不在安全边界范围内；除非报告证明存在真正的权限边界绕过，或说明 pi 如何授予了本地用户原本不具备的访问权限。
